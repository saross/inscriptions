## Failure modes — avoid (cumulative across the project)

- **Path typos in agent briefs.** Sapphire workdir is `~/Code/inscriptions`.
- **`pgrep -f` self-match.** Use bracket-escape: `pgrep -f "[.]venv/bin/python3.*foo.py"`.
- **Agent stalling on inline script streaming.** Long Python / R scripts via Write tool's `content` parameter; never stream as chat (600 s watchdog).
- **Monitor loops on stale files.** Use mtime or fresh-run markers, not file existence.
- **Zotero FTS does not index DOI field.** Use locally-built DOI index for idempotency.
- **Agent silent-parameter-reduction** (critical-friend gate failure pattern). Hard-stop rules in briefs.
- **Background-agent monitor-and-exit pattern.** Agents that arm a Monitor and exit don't re-fire from monitor events. Use Bash run_in_background with `until ! kill -0 PID` for "wait for PID death" patterns.
- **baorista API gotchas** (preserved from smoke-test iterations 2026-05-03):
  - `timeRange` must be descending (LARGER first).
  - `(upper - lower + 1) %% resolution == 0` required.
  - Per-event col1 > col2 (numeric ordering, NAMES are conventional).
  - Every event must satisfy `lower <= col2 <= col1 <= upper`.
  - `expfit` returns S3 `fittedExp` with `$rhat` / `$ess` directly — no `$samples` slot.
- **Sapphire fresh-R-install user-library bootstrap** (preserved from baorista install fix 2026-05-04): without `R_LIBS_USER` directory creation + `.libPaths()` push, `install.packages()` falls back to root-owned `/usr/local/lib/R/site-library` and crashes within seconds.
